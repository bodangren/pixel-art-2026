# Spec: Static Benchmark/Review App for Game-Asset Model Leaderboard

## Summary
Transform the `pixel-art-benchmark` repository into a semi-automated benchmark system for evaluating image models on generating starter pixel-art asset packs for games in the `advantage-games` repository.

## Core Requirements
- **Next.js Static Export**: The app must be exportable to a static site for public viewing.
- **Modes**: 
    - `NEXT_PUBLIC_REVIEW_MODE=true`: Local/dev mode with editable rubric controls and save/export features.
    - `NEXT_PUBLIC_REVIEW_MODE=false`: Public read-only mode for the benchmark and leaderboard.
    - **Blind Review Toggle**: An option in Review Mode to hide the `model_id` during scoring to prevent brand bias. Model ID is revealed upon save.
- **Data Model**:
    - **Runs**: `data/runs/<run-id>/run.json` (metadata) and `data/runs/<run-id>/review.json` (mutable scores).
    - **Assets**: Image files stored in `data/runs/<run-id>/assets/`.
- **Dynamic Game Targeting**: Assets generated will vary based on the specific game selected from the `advantage-games` repository. The preview UI must dynamically render available sheets and backgrounds based on `run.json` manifests, rather than hardcoded file names.
- **Generation Method Agnostic**: Models can generate assets via any pixel-art method (Python/Pillow, Node/Canvas, HTML/CSS, SVG, etc.), provided the output does not rely on generative image models (like DALL-E). This method should be tracked in the metadata.
- **Leaderboard**: 
    - Aggregated scoring across models and runs.
    - **Rankable Metrics**: The leaderboard must allow sorting by overall score AND individual rubric subscores (e.g., "Best Animation Alignment").
    - **Golden Samples**: Ability to flag specific runs as "Golden Samples" (perfect 5.0 reference runs) and pin them to the top of the leaderboard as visual anchors to prevent scoring drift.

## Technical constraints
- No database; JSON-in-repo is the source of truth.
- Static export compatibility is mandatory.
- Use Tailwind CSS for rugged, high-contrast styling.

## Routes
- `/leaderboard`: Global rankings (sortable by subscore) with pinned Golden Samples.
- `/models/[modelId]`: Model history and performance.
- `/runs/[runId]`: Detailed run analysis and review interface.

## Asset Inspection Requirements
- **Automated Technical Linter**: A pre-review script that analyzes assets for grid alignment (e.g., 64x64 tiles exactly centered in a 3x3 sheet), strict palette consistency across the pack, and true 0% alpha transparency. Outputs a "Technical Grade" appended to `run.json`.
- **Interactive Previews**: Specialized previews for sprites (animations, frame-stepping, zoom, grid overlays) and backgrounds.

## Rubric Categories (Sortable on Leaderboard)
- **Background**: Readability, Style Fit, Appeal
- **Sprite Sheets**: Silhouette, Animation Alignment, Style Fit
- **Whole Pack**: Coherence, Game Fit, Prototype Readiness (Boolean)
- **Technical Grade**: Generated by the automated linter (Alignment, Palette, Transparency)
