# Pixel Art Benchmark Rules

## PURPOSE
This is a benchmark test to evaluate AI model's ability to generate pixel art game assets. The test measures creativity, adherence to specifications, and attention to detail.

## STRICT PROHIBITIONS

1. **NO REFERENCE TO OTHER MODELS' WORK**
   - Do NOT read or reference scripts, code, or assets generated by other AI models
   - Do NOT look at files in directories like `Haiku-4.6/`, `Gemini-2.5-flash/`, `gpt-5.3-codex-medium/`, etc.
   - Do NOT examine `generate_assets.py` or any other reference implementations
   - This is NOT a coding exercise - this is a creative asset generation benchmark

2. **NO COPYING OR DERIVATIVE WORK**
   - All assets must be generated from scratch based solely on `pixel-art-benchmark.md`
   - Do not copy structures, approaches, or patterns from other models
   - Each model's output should be unique and original

3. **FAIR EVALUATION**
   - The benchmark compares original creative output, not copy-paste ability
   - Evaluation is based on: visual quality, adherence to specs, originality
   - Cheating invalidates the benchmark results

## ACCEPTED APPROACHES

- Read ONLY the specification file: `pixel-art-benchmark.md`
- Use any tools or techniques available (Python PIL, JavaScript Canvas, etc.)
- Generate assets based purely on the spec requirements
- Be creative and original in your implementation

## OUTPUT REQUIREMENTS

All files must be placed in the root of the `glm-4.7/` directory:
- `background.png` (390x700)
- `hero-3x3-sheet.png` (192x192)
- `goblin-3x3-sheet.png` (192x192)
- `orb-sheet.png` (144x48)

## CONSEQUENCES OF CHEATING

Any violation of these rules results in:
- Automatic disqualification from the benchmark
- Invalidated test results
- Loss of credibility for the model

## SIGNATURE

By proceeding with this benchmark, you acknowledge that you understand and agree to these rules.
